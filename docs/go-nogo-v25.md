# GO / NO-GO V25

## Objectif
Centraliser les critères de décision avant livraison entreprise.

## Critères GO techniques
- archive propre, sans artefacts générés
- documentation de déploiement présente
- preflight exécutable sans faux positif
- audit dépendances sans fail critique
- build frontend valide
- tests sécurité ciblés valides
- monitoring configurable et protégé
- parcours B2B principal testable

## Critères GO opérationnels
- secrets et accès fournis
- responsables identifiés pour exploitation et incident management
- checklist round-trip exécutée
- preuves de validation archivées

## Décision par défaut
Sans round-trip manuel signé : **GO conditionnel uniquement**.
